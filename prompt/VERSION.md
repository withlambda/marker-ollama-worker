# Context
This file, `VERSION`, is a single-line text file that stores the current version of the project (e.g., `1.10.2`).

# Logic
It is read and updated by the `release.sh` script during the automated version bumping and tagging process. It ensures a single source of truth for the project's version across different components.

# Goal
The prompt file provides the structure and purpose of the version file, allowing it to be recreated as part of the project's metadata.
